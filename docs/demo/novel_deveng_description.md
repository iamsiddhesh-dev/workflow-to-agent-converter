Our CI runs a big test suite and some tests are just flaky, not real breaks. When a build fails we want something to look at which tests failed, check if they've been flaky before (we keep a rough history), and label the failure as "flaky, safe to retry" or "looks like a real regression, needs a human." Real regressions should ping the owning team based on which test file broke. We want a weekly digest of how many failures were flaky vs real so we can tell if flakiness is getting worse.
